<objective>
Fix test creation API endpoint connection failure

The test creation form is failing with ERR_CONNECTION_REFUSED when trying to POST to `/api/v1/test-definitions/`. The issue is that the frontend is using `window.location.origin` (which resolves to `http://localhost:8080`) but the API call is failing because:

1. The backend service runs on port 8011 internally
2. Nginx on port 8080 should proxy `/api/v1/` requests to the backend
3. The API endpoint URL may be malformed or missing a trailing slash

This plan will diagnose and fix the API endpoint configuration to ensure proper routing through nginx.

Purpose: Enable test creation functionality in the frontend
Output: Working test creation API calls
</objective>
