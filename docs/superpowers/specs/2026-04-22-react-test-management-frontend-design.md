# React Test Management Frontend Design

**Date:** 2026-04-22  
**Type:** Feature Design  
**Status:** Approved

## Overview

Design a minimal React-based frontend for internal team management of test cases. The app focuses on essentials: create tests, view test list, and trigger test runs. No authentication, no advanced features - simple and functional.

## Requirements

### Functional Requirements

1. **Test List View**
   - Display all tests from Test Case Service API
   - Show test name, description, status, last run time
   - Support search and tag filtering
   - Provide "Run" button for each test

2. **Test Creation**
   - Single-page form with all fields
   - Fields: name, description, test_id, url, environment variables, test steps
   - Dynamic step editor (add/remove steps)
   - Support 4 step types: navigate, click, fill, wait

3. **Test Execution**
   - Trigger test run via Scheduler Service API
   - Fire-and-forget approach (no status polling)
   - Show success message with job ID

### Non-Functional Requirements

- **No authentication** - Development/internal use only
- **Simple state management** - React hooks only (no Redux/Zustand)
- **Fast development** - Use Vite for quick build
- **Clean integration** - Work with existing FastAPI backends

## Data Model

### Test Object

```javascript
{
  id: number,                    // Auto-generated by backend
  name: string,                  // Required: "Login Test"
  description: string,           // Optional: "Verify user login"
  test_id: string,               // Optional: "login-001"
  url: string,                   // Required: "https://example.com/login"
  environment: object,           // Optional: {"USERNAME": "test", "PASSWORD": "123"}
  tags: string[],                // Optional: ["smoke", "critical"]
  test_steps: Step[],            // Array of test steps
  created_at: string,            // ISO timestamp
  updated_at: string             // ISO timestamp
}
```

### Step Object

```javascript
{
  id: number,                    // Auto-generated
  step_type: string,             // "navigate" | "click" | "fill" | "wait"
  selector: string,              // Optional: CSS selector for click/fill
  value: string,                 // Optional: URL for navigate, text for fill
  order: number                  // Step order (1, 2, 3...)
}
```

## API Integration

### Endpoints Used

**From Test Case Service (http://localhost:8011):**

```javascript
// Get all tests
GET /api/v1/test-definitions
Response: { items: Test[] }

// Create test
POST /api/v1/test-definitions
Body: {
  name: string,
  description?: string,
  test_id?: string,
  url: string,
  environment?: object,
  tags?: string[]
}
Response: Test

// Add steps to test
POST /api/v1/test-steps/test-definition/{test_id}
Body: {
  step_type: string,
  selector?: string,
  value?: string,
  order: number
}
Response: Step
```

**From Scheduler Service (http://localhost:8012):**

```javascript
// Run test
POST /api/v1/jobs
Body: {
  test_definition_ids: number[]
}
Response: {
  job_id: number,
  status: "pending"
}
```

### API Client Implementation

```javascript
// src/api.js
const TEST_API = 'http://localhost:8011/api/v1';
const SCHEDULER_API = 'http://localhost:8012/api/v1';

export const getTests = async () => {
  const response = await fetch(`${TEST_API}/test-definitions`);
  if (!response.ok) throw new Error('Failed to fetch tests');
  return response.json();
};

export const createTest = async (testData) => {
  const { test_steps, ...testData } = testData;
  
  // Create test first
  const testResponse = await fetch(`${TEST_API}/test-definitions`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(testData)
  });
  if (!testResponse.ok) throw new Error('Failed to create test');
  const test = await testResponse.json();
  
  // Then add steps
  for (const step of test_steps) {
    await fetch(`${TEST_API}/test-steps/test-definition/${test.id}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ ...step, order: step.order })
    });
  }
  
  return test;
};

export const runTest = async (testId) => {
  const response = await fetch(`${SCHEDULER_API}/jobs`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ test_definition_ids: [testId] })
  });
  if (!response.ok) throw new Error('Failed to run test');
  return response.json();
};
```

## Error Handling

### API Errors

- Show user-friendly error messages
- Don't crash on API failures
- Log errors to console for debugging

```javascript
const handleSubmit = async () => {
  try {
    setLoading(true);
    await createTest(formData);
    alert('Test created successfully!');
    // Refresh test list
    loadTests();
  } catch (error) {
    alert(`Failed to create test: ${error.message}`);
  } finally {
    setLoading(false);
  }
};
```

### Form Validation

- Required fields: name, url, at least one step
- Validate URL format
- Validate step configuration (selector required for click/fill)

```javascript
const validateForm = () => {
  if (!formData.name.trim()) {
    alert('Test name is required');
    return false;
  }
  if (!formData.url.trim()) {
    alert('URL is required');
    return false;
  }
  if (formData.test_steps.length === 0) {
    alert('At least one test step is required');
    return false;
  }
  return true;
};
```
